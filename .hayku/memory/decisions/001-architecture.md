# 決策 001：核心架構與起步策略

## 決策

採用方案 A（由內而外），以 Claude CLI 為基底改造：
1. 先建立 Memory + Tool Manager 核心
2. 用 `.hayku/` 目錄作為 Agent 的運作核心，放在專案根目錄
3. 透過 `CLAUDE.md` 定義 Agent 行為
4. 碰到 CLI 天花板時再抽出去用 Agent SDK 替換

## 原因

- Claude CLI 已具備記憶、工具執行、多 Agent 派遣等基礎能力
- 不需從零開始，今天就能運作
- `.hayku/` 跟著專案走，git 可追蹤，部署時記憶跟著走
- 漸進式演進比一次到位更務實

## 替代方案（未選擇）

- 方案 B（由外而內）：先做 MVP 再強化 — 風險是核心不夠紮實
- 完全自建：用 Agent SDK 從零開始 — 工作量太大，初期不必要
